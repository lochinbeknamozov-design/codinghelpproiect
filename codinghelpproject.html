What is HTML?

HTML (Hypertext Markup Language) is a coding language used to for the creation of online websites using a series of different commands and functions in order to write and format infomation.

This would make HTML a type of Markup Language which is a standard of text-encoding (assigning to numbers to characters ex. A,B,C 1,2,3, !,?,:) Markup languages use their characters and overall language to allow for writing text that can then be used to convey specific information. Other notable examples of Markup Languages are XML, useful for sharing data and SVG, used for vector graphics.

HTML is a Markup language that focuses on making web pages that can be presented to other people through online web browsers. A person who knows the language and tools of HTML can write HTML files that can then by stored on servers. HTML files can then be called by web browsers, which open, convert the text inside the file into a more visual medium, and then display HTML documents on to a web page. Inside HTML, there are many different elements that have their own formatting and functions. A element can be something like a title, image, or text box on the html page. These elements and the functions that manipulate them can be expanded if using other languages such as CSS (Cascading Style Sheets) and JavaScript.

How do I start coding in HTML?

To first start coding in general make sure that you have some sort of text editing software. I would recommend:

  • Notepad++ : Very minimalistic source code editor, good for straight to the point coding
  • Mircosoft Visual Studio Code : Industry standard for coding, lots of support for other languages and plugins, very customizable.

Both of those were used to make this website. Here is some other other notable text editors

  • Geany
  • Emacs Series
  • Sublime Text
  • Vim

Once you have your text editor of choice and after set up anything you have to for the text editor you should be free to create a text file. Name it something important to the website, you want to make so you can save it for later, and then put that file into a folder. Then change the file extension from .txt to .html. This creates an HTML document that you can now open in a web browser.

The document you have made will have nothing on it because we haven’t even started writing it out. So go back to the file in the folder and open it up with the text editor. For this example, I will use Notepad++. The first thing you want to do before writing the website is to place <!DOCTYPE html> in the first line, this declares your file to be truly a html document. From there you want to create your “head”. To do this in the next line write in <head> and then below that write in <title> and write the page title you want the website to have. Here's mine, for example.

<!DOCTYPE html>
<html> 
		<head> 
			<title> Coding Help by Loach </title> 
			<link rel="stylesheet" href="css/main.css"> 
			<meta charset="UTF-8"> 
			<meta name="viewport" content="width= device-width, initial-scale=1.0"> 
		</head>

To figure out what is going on here I will try to explain what the <head> tag means. The <head> tag is used to archive important data for the rest of the document, with the most notable being the <title>, which is required for all HTML websites. Here I have also linked the HTML document to my css file, which will be important for later. Then I also have the <meta> tag, an all-purpose tag used here to indicate the character set I am using; almost all websites use UTF-8. The meta tag is also used to change the size of the screen if you’re on a phone to make things more accessible.

The question after this is, what is a tag? A tag is anything using < >, like <p> or </p>, these indicate the space of an element on the website, so here <p> says that there is going to be code for a paragraph and </p> says that the paragraph code has ended. Anything between those two tags are a part of that paragraph. The best way to visualize it is like a box, anything in the <p> and <p> is in that box, while the box can also have other boxes inside of it like <u> </u>, which indicates an underlined piece of text. Other tags don’t have these ending tags like </p>; instead, the code used is placed alongside the tag. In the previous image, you can see it with the <link and meta tags with all the code called being inside the < >.

Tags can also have different attributes such as IDs which differentiate one tag from another if they are the same kind of tag. Another attribute are styles which allow for more customization of a tag, for example here is a text box for the tile.

Here, the title has the style and id attribute written inside the tag space. Id simply gives that element a name here being title. The style calls for the element's, here a header (h1), line height and font family. These two in particular manipulate the distance between lines of text vertically and the font of the text.

The line under that has a tag of title2 which is a custom tag. Custom tags if not given any styling default to being text which reflects how a HTML page is based off of things written on a text document. To make this tag have any distinct features from a simple paragraph tag you first need to get a hold of your CSS file. If you remember from text on the head tag I linked my HTML file to my CSS file. This allows me to take advantage of what I wrote on that CSS file. The CSS file should defenityly be close to where it's HTML counterpart is ideally, you should have a folder containing your HTML file and CSS file together. Now, to explain what CSS is after we have made sure it existed by creating a file and changing its name into .CSS and then made sure it was linked to the HTML file. CSS is a very necessary suplementary system to HTML. This is the source of all the unique styles you can put on to a tag. You can make a tag have a specific style on a HTML file but a CSS folder is more reilable and organized in that it allows you to look at all of your unique styles and tags.
